# 效率功能
features.efficiency.title = 带宽减少约 70%
features.efficiency.subtitle = 矿池和矿工均可受益
features.efficiency.body = SV1 的 JSON 消息占用大量带宽，而 SV2 使用精简的二进制格式，降低 CPU 负载，将矿池带宽使用减少约 60%，矿工带宽减少约 70%，降低矿池运营成本，帮助带宽受限的矿工更稳定地运行。

# 安全功能
features.security.title = 安全性
features.security.subtitle = 保护收益
features.security.body = Stratum V2 通过加密与认证保护矿工与矿池之间的通信，抵御中间人攻击（例如替换矿池凭证以劫持算力并重定向作业）。加密传输也让 ISP 或实施大规模路由攻击的对手更难窃听或篡改流量。SV2 还允许矿工自行构建区块模板，提高透明度并消除带外支付。

# 盈利能力功能
features.profitability.title = 利润提升高达 7.4%
features.profitability.subtitle = 在实际部署中得到验证
features.profitability.body = Stratum V2 的技术升级带来可量化的收益提升：更快的模板下发与更低的延迟减少无效工作，让每个区块捕获更多交易手续费；再结合端到端加密与更快的区块传播等改进，利润最高可提升 7.4%。
features.profitability.link = 阅读案例研究 →

# 隐私功能
features.privacy.title = 隐私改进
features.privacy.subtitle = 防止 ISP 监听
features.privacy.body = Stratum V2 对挖矿流量进行加密，避免互联网服务提供商（ISP）等网络中介看到明文份额与消息。在 Stratum V1 中，这些数据可能被用来推算矿工算力，并据此估算收益。

# 自主权功能

features.autonomy.subtitle = 矿工选择区块内容
features.autonomy.body = 当前少数矿池主导着比特币区块空间，部分矿池甚至充当其他矿池的代理。矿池可以选择性地包含或排除交易，带来潜在的审查风险。Stratum V2 使矿工能够自行构建区块模板，将其从被动算力提供者转变为维护比特币抗审查性的积极参与者。
